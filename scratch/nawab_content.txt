Abstract
Livestock is a valuable sector of the agricultural economy, and currently the purchasing and selling of animals is not done efficiently due to lack of transparency in the process. Moreover, farmers and breeders have to contend with non-uniform prices for their livestock, uncertainty with regard to safety during purchasing and selling, as well as regional shortages and surpluses. In AnimalMarket, we propose an effective web-based platform for the efficient marketing of livestock, primarily aimed at guaranteeing a convenient and above all a safe mode of trading animals over the internet. We intend to foster closer links between remote and deprived farmers and larger markets, both local and further afield, through the innovative use of modern communication and information technologies, to facilitate and consequently improve the buying and selling process between farmers and vendors, while ultimately promoting the procurement of quality animals, and thus ensuring informed and credible animal marketing.
This application is built with a full stack approach using React.js as the frontend and FastAPI (Python) as the backend. This application has real-time chat functionality implemented using WebSockets for communication between buyer and seller. Also it uses JWT for authentication and a dynamic review system for user rating. The Price Estimator is another exciting feature of this application. It uses a Machine Learning model that is trained with the historical market data of various animals. Using this model the price of an animal can be estimated based on certain attributes of the animal such as the weight, breed and age of the animal in order to minimize price variance.
We’ve built a system that can handle thousands of users online at the same time, one on one real time messaging without too much delay and proper image handling of the listings. Our pricing model is also yielding reasonably accurate information for negotiable prices for the items. Overall we believe that AnimalMarket will serve to improve the efficiency of livestock trading. With a web-based system that allows farmers to buy and sell livestock through a simple, data-rich and safe interface, that ensures truthful pricing of the items and hence that the marginalisation of small scale and resource poor farmers is eliminated, while at the same time making the buyer’s experience as smooth as possible.

4.2 System Modules and Implementation
The AnimalMarket application is made up of three parts. Each part talks to the others using API endpoints or WebSocket channels.
4.2.1 Authentication and User Management Module
Security is very important when it comes to buying and selling things online. This part of the application handles user registration and session management.
Implementation: The backend uses the OAuth2PasswordBearer flow. When a user logs in the server creates a token called a JSON Web Token or JWT for short. This token is signed with a key using the HS256 algorithm.
Data Flow: This token is stored in the frontend’s Local Storage and attached to the header (Authorization: Bearer <token>) of every subsequent HTTP request. This ensures that protected routes (e.g., POST /animals/, GET /users/me) are accessible only to authenticated users.
4.2.2 Marketplace and Search Module
This part of the application handles the business logic of the AnimalMarket platform. This includes things like getting lists of animals filtering those lists and handling images.
Image Handling: When a user uploads an image of an animal it is sent to the backend using something called FormData. The backend cleans up the filename to prevent any security problems saves the file to a place and stores the path to the file in the MySQL database.
Search Logic: The filtering system is built using something called dynamic SQLAlchemy queries. The user can choose to filter by things, like price, city and breed. The backend then builds a query based on what the user chose which makes it faster to get the right data.
4.2.3 Real-time Communication Module
To make it easy for people to talk to each other away a chat system was made using WebSocket.
Connection Manager: A special class called ConnectionManager was made in Python. It keeps track of all the people who are connected and their WebSocket objects.
Message Routing: When one person wants to send a message to another person the manager checks if the other person is online. If they are the message is sent away. If they are not the message is. Sent when they log in next.

4.3.3 Data Preprocessing
The data from the market can be messy. So some things were done to clean it up before training the model:
Missing Value Imputation: If some important information was missing like the weight or price that animal was not used. If the age was missing it was guessed using the age of that kind of animal.
Outlier Detection:  If an animal had information like being 1000 kilograms it was not used because it would mess up the model.
Feature Encoding: The model needs numbers so the breed and color were changed into numbers using One-Hot Encoding.
Encoding, creating binary columns for each category (e.g., Breed_Beetal: 1, Breed_Teddy: 0).
4.3.4 Model Configuration
The Random Forest Regressor was selected for the final implementation.
Hyperparameters:
n_estimators (Number of trees): 100
random_state: 42 (To ensure consistent results across runs)
max_depth: None (Allowing nodes to expand until all leaves are pure or contain less than min_samples_split samples).
Training Split: The data was split into two parts, 80% for training and 20%, for testing.

7.1 Conclusion
The livestock sector is very important for the economy. However it has been facing problems like inefficiencies and lack of transparency for a time. Also there are many intermediaries involved. The main goal of this thesis was to create AnimalMarket, a web application that can solve these problems. AnimalMarket is a web application that uses advanced web technologies and Artificial Intelligence to provide a complete solution for buying and selling livestock.
The development process used a set of technologies, including React.js for creating a user-friendly interface and FastAPI for a fast and efficient backend. This architecture made it possible to create a Single Page Application that works well on both desktop and mobile devices making it accessible to people in rural areas. In conclusion AnimalMarket shows that it is possible to bring about transformation in the livestock sector and that it can be very beneficial.
